## ūüďĆ What Is This?

**DevOps CLI** is a command-line automation tool that replaces repetitive terminal work with clean, structured commands. Instead of manually running scripts, checking system resources, or hunting through log files ‚ÄĒ this tool does it all in one place.

This is **Project 1** of my Python learning journey ‚ÄĒ built step by step to master Python fundamentals, standard library modules, and real-world CLI development patterns.

---

## ‚ö° Quick Start

```bash
# 1. Clone the repo
git clone https://github.com/CaptainAni187/DEVOPS-CLI.git
cd DEVOPS-CLI

# 2. Create and activate virtual environment
python3 -m venv venv
source venv/bin/activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Run your first command
python3 cli/main.py status
```

## ūüĖ•ÔłŹ Commands

```
python3 cli/main.py <command> [options]
```

| Command | Options | What it does |
|---|---|---|
| `init` | ‚ÄĒ | Creates `logs/` `backups/` `temp/` `data/` folders |
| `status` | ‚ÄĒ | Project config + live CPU / RAM / Disk snapshot |
| `monitor` | `-n N` `--parallel` | Live system resource monitoring |
| `run` | `<cmd> [args]` | Run any shell command and capture output |
| `backup` | `[file]` | Backup a file with a timestamp in the name |
| `analyze-logs` | `[file]` | Parse log file ‚ÄĒ count ERROR / WARNING / INFO |
| `env` | ‚ÄĒ | Show `.env` variables (secrets auto-redacted) |

### ūüď¶ Data Structures & OOP ‚ÄĒ `models/config_model.py`

```python
from dataclasses import dataclass
from enum import Enum

class LogLevel(Enum):
    INFO = "INFO"
    ERROR = "ERROR"

@dataclass
class Config:
    project_name: str
    log_level: LogLevel = LogLevel.INFO
```

| Concept | What I learned |
|---|---|
| `@dataclass` | Auto-generates `__init__`, `__repr__` ‚ÄĒ no boilerplate |
| `Enum` | Named constants that prevent typos ‚ÄĒ `LogLevel.ERROR` not `"ERORR"` |
| Type hints | `str`, `int`, `Optional[str]` ‚ÄĒ documents what functions expect |
| `__post_init__` | Runs after `__init__` ‚ÄĒ converts raw strings to enum values |

### ūüĒć Regex, Generators & Comprehensions ‚ÄĒ `core/log_analyzer.py`

```python
import re
from collections import Counter

LOG_PATTERN = re.compile(r"\b(ERROR|WARNING|INFO|DEBUG)\b")

def stream_log_lines(path):       # Generator ‚ÄĒ one line at a time
    with open(path) as f:
        for line in f:
            yield line

errors = [line for line in lines if "ERROR" in line]   # List comprehension
```

| Concept | What I learned |
|---|---|
| `re.compile()` | Compile regex once, reuse in loops ‚ÄĒ faster than inline patterns |
| `\b(ERROR\|WARNING)\b` | Word boundaries prevent `INFORMATIONAL` matching `INFO` |
| `match.group(1)` | Extract the captured group from a regex match |
| `yield` generator | Produces one item at a time ‚ÄĒ memory-efficient for huge files |
| `collections.Counter` | Dictionary built for counting ‚ÄĒ missing keys return `0` not `KeyError` |
| List comprehension | `[x for x in items if condition]` ‚ÄĒ filter and transform in one line |
| Dict comprehension | `{k: v for k, v in pairs}` ‚ÄĒ build dicts with filtering inline |

### ūüĖ•ÔłŹ System Monitoring & Concurrency ‚ÄĒ `core/system_monitor.py`

```python
import psutil, threading

def monitor_loop():
    cpu = psutil.cpu_percent(interval=1)
    ram = psutil.virtual_memory().percent

t1 = threading.Thread(target=monitor_loop, daemon=True)
t2 = threading.Thread(target=analyze_logs)
t1.start(); t2.start()
t1.join();  t2.join()
```

| Concept | What I learned |
|---|---|
| `psutil` | Read real CPU, RAM, Disk, process count from the OS |
| `datetime.strftime()` | Format timestamps ‚ÄĒ `%Y-%m-%d %H:%M:%S` ‚Üí `2026-03-17 10:22:01` |
| `threading.Thread` | Run two functions simultaneously in parallel |
| `t.start()` / `t.join()` | Launch a thread / wait for it to finish |
| `daemon=True` | Thread dies when main program exits ‚ÄĒ prevents hanging |
| Shared dict | Pass a mutable dict into a thread to receive results back |
